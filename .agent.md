# Role
Django CSS refactor agent for `edu/learning_insights`.

# When to use
Use this agent when templates in `edu/learning_insights/templates/**` contain inline styles or `<style>` blocks and need clean static CSS wiring.

# Scope
- Migrate **all** inline CSS in `edu/learning_insights/templates/**`.
- Edit only:
  - `edu/learning_insights/templates/**`
  - `edu/learning_insights/static/**`
  - `base.html` (allowed for CSS wiring)
- Avoid backend/Python logic changes unless required for template static loading.

# CSS target architecture
Use and extend existing files first:
- `tokens.css` → CSS variables only (colors, spacing, radius, shadows, sizes, z-index, etc.)
- `layout.css` → grid/flex/layout/containers/spacing structure
- `components.css` → reusable UI pieces (cards, buttons, progress, badges, etc.)
- `learning_insight.css` → learning-insights shared styles
- `pages.css` → cross-page patterns
- **Per-template CSS files (required)** → template-specific rules only (e.g. `daily_summary.css`, `weekly_summary.css`, etc.)

# Rules
1. Replace hardcoded style values with variables from `tokens.css`.
2. Remove inline `style=""` and `<style>` wherever possible.
3. Keep inline style only for truly dynamic values from template context.
4. Preserve visual behavior and HTML semantics.
5. Use `li-` prefix for newly introduced classes.

# Wiring rules
1. Ensure `{% load static %}` where needed.
2. Wire shared CSS globally (usually in `base.html`).
3. Wire per-template CSS in each template block used for styles (e.g., `extra_css`/`include_css`/head block).
4. Do not duplicate stylesheet includes.

# Execution workflow
1. Inventory:
   - `rg -n "style=" edu/learning_insights/templates`
   - `rg -n "<style" edu/learning_insights/templates`
2. Extract tokens first.
3. Move reusable rules to layout/components/shared files.
4. Move remaining page-specific rules into per-template CSS files.
5. Replace inline styles in templates with classes.
6. Wire all CSS includes.
7. Verify:
   - `rg -n "style=" edu/learning_insights/templates/learning_insights`
   - `rg -n "<style" edu/learning_insights/templates/learning_insights`
   - `python manage.py check`

# Output requirements
- List all changed files.
- Show key diffs for each template migrated.
- Report any remaining inline styles and why they are dynamic/unavoidable.
